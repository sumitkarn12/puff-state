📚 Complete API Reference Manual

createPuffStore(name, config)

Initializes a global decoupled storage client instance bucketed under an explicitly isolated namespace wrapper.

Accepted Arguments:
  • name (String, Optional): The persistence lookup namespace key. Defaults to "puff-store".
  • config (Object, Optional): Core configuration overrides. Supports:
    • defaultExpiry: Fallback key TTL lifespan in milliseconds (Default: 365 days).
    • storage: Underlying persistent Web Storage API engine target (Default: localStorage).
Code Usage Example:
const store = createPuffStore("app-cache", {
  defaultExpiry: 60 * 60 * 1000, // 1 hour default
  storage: sessionStorage        // Use sessionStorage instead
});

store.init()

Hydrates and syncs all non-expired structural variations out of hardware persistent layers back into active in-memory Maps.

Accepted Arguments:

None.

Code Usage Example:
// Invoke immediately on application startup bootstrapping
store.init();

store.setState(key, value, dontFire, ttl)

Commits a runtime variable structure updates into memory and persistent state layers, automatically executing reactive hooks.

Accepted Arguments:
  • key (String): Target state property identifier.
  • value (Any): Serialized data parameters payload.
  • dontFire (Boolean, Optional): Set to true to perform quiet database mutations without alerting active observers.
  • ttl (Number, Optional): Lifespan limit overrides designated in milliseconds.
Code Usage Example:
// Stores values reactively that self-delete in exactly 10 seconds
store.setState("notification", "Logged in!", false, 10000);

store.getState(key)

Queries memory layers for corresponding values. If timestamps have lapsed past limits, data vanishes and returns null.

Accepted Arguments:
  • key (String): The lookup property identity.
Code Usage Example:
const activeTheme = store.getState("user_theme"); 
// Returns value or null if expired/non-existent

store.subscribe(key, callback)

Attaches a callback function triggered reactively every time a target key updates value patterns via state methods.

Accepted Arguments:
  • key (String): Target state key property to actively map listeners onto.
  • callback (Function): Executable execution chain signature: (newValue, oldValue) => void.
Code Usage Example:
const unsubscribe = store.subscribe("user_theme", (next, prev) => {
  document.body.className = next;
});

// Execute whenever cleaning up event handlers to prevent memory leaks
// unsubscribe();

store.keys()

Retrieves an array representation mapping out all functional active keys recorded across storage engines.

Accepted Arguments:

None.

Code Usage Example:
console.log(store.keys()); // e.g., ["user_theme", "auth_token"]
